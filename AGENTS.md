# Project Instructions

## Goals
- Read repository docs before making code or document changes.
- Ask for confirmation before applying file edits, deleting files, or running commands that may change the workspace in a meaningful way.

## Required Reading
- Read the main implementation change note under `docs/` that matches `w1_w2*.md` first when the task touches implementation or requirements.
- Read `docs/w1.md`, `docs/w2.md`, `docs/w3.md`, and `docs/w4.md` when the task involves project notes or staged work items.

## Working Rules
- Treat the files under `docs/` as the primary project context unless the user explicitly overrides them.
- Summarize the intended change before editing files.
- Prefer minimal changes that match the existing repository structure.
- If requirements from different docs conflict, stop and ask the user which source should win.
