## 3. W17 设计与实现

### 3.1 配置热更新与回滚流程
新增 infra 发布接口：
1. `publish_config_snapshot(snapshot)`：发布新版本快照（仅接受更高版本）。
2. `publish_config_patch(patch, applied_version)`：在当前快照上打补丁并自动 `version+1`。
3. `rollback_config_to(version)`：回滚到历史版本。
4. `current_config_version()`：读取当前版本号。

说明：
1. 仍复用既有 `ConfigRepository` 历史链路，回滚语义与 W1 一致。
2. 发布/回滚接口在同一全局状态锁下执行，保证流程串行化。

### 3.2 热更新生效路径
`ReactorHttpServer` 增加动态限流刷新：
1. 通过 `refresh_rate_limit_from_config_if_needed()` 按配置版本更新生效值。
2. `allow_request_now()` 在判定前触发版本检查。
3. 配置版本变更后自动重置限流窗口计数，确保新策略立即生效。

结果：
1. 不需要重启服务即可调整 `gateway.rate_limit.qps/burst`。
2. 已支持“低限流 -> 发布补丁 -> 高限流”在线切换。

### 3.3 灰度发布与回滚
灰度比例改为“Header 优先 + 配置兜底”：
1. Header：`x-gray-percent`
2. Config：`rpc.gray.percent`

行为：
1. Header 存在时按 Header。
2. Header 缺失时按 `rpc.gray.percent`。
3. 回滚配置版本后灰度策略自动回退。

### 3.4 TLS/mTLS 与证书轮转策略
`invoke_tcp` 增加 TLS 分支（Boost.Asio SSL）：
1. 配置 `net.tls.enabled=1` 启用 TLS。
2. 配置 `net.tls.mtls.enabled=1` 启用双向认证。
3. 支持 `ca_file/cert_file/key_file/server_name`。

证书轮转策略：
1. 缓存 SSL context。
2. 当配置版本、证书路径或文件 mtime 变化时触发重建。
3. 若重建失败且旧 context 可用，则保留旧 context 继续服务（避免瞬时配置错误导致全量失败）。

新增可观测：
1. `tls_runtime_snapshot()` 返回：
- `tls_enabled`
- `mtls_enabled`
- `loaded_config_version`
- `context_reload_count`
- `context_reload_failures`

## 5. 运维手册

### 5.1 限流（在线调整）
目标：不重启服务在线调参。

建议流程：
1. 读取当前版本：`current_config_version()`。
2. 发布补丁：
- `gateway.rate_limit.qps`
- `gateway.rate_limit.burst`
3. 观察 `429` 比例与入口成功率变化。
4. 异常时回滚到锚点版本：`rollback_config_to(anchor_version)`。

### 5.2 熔断（参数建议）
常用键：
1. `rpc.circuit.failure_threshold`
2. `rpc.circuit.open_ms`
3. `rpc.circuit.half_open_success`
4. `rpc.circuit.half_open_max_probes`

建议：
1. 先小步调整 `open_ms` 和 `failure_threshold`。
2. 观察 `blocked_by_circuit_breaker` 与失败率。
3. 若误熔断增多，回滚到上一个稳定版本。

### 5.3 回滚演练（分钟级）
标准流程：
1. 演练前记录 `anchor_version`。
2. 发布目标变更（灰度、限流、TLS 参数）。
3. 注入验证流量，确认变更已生效。
4. 执行 `rollback_config_to(anchor_version)`。
5. 校验关键指标恢复（成功率、429 比例、灰度命中、TLS 握手成功率）。

SLO 建议：
1. 回滚操作 + 生效确认控制在 2 分钟内。

### 5.4 证书故障排查
常见问题：
1. `tls_handshake_failed:*`：证书链或 mTLS 验证失败。
2. `tls_context_unavailable:*`：证书文件不可读或格式错误。
3. 握手间歇性失败：轮转时新证书加载失败。

排查步骤：
1. 检查配置项路径是否存在且权限可读。
2. 核对 `ca_file` 与 server/client 证书签发链是否一致。
3. 读取 `tls_runtime_snapshot()`：
- `context_reload_failures` 是否增长。
- `loaded_config_version` 是否已切到目标版本。
4. 若失败持续，立即回滚证书配置版本。
