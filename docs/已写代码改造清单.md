# 已写代码改造清单（对应开发指南周计划升级）

## 1. 文档目的
本文件用于明确“当前仓库已落地代码”中需要继续改造的部分，优先覆盖以下升级方向：
1. 协程调度器升级为三级调度 + 工作窃取。
2. 全链路可观测性升级为 Tracing + Metrics + Logging 一体化。
3. 负载均衡升级为自适应策略并接入熔断/降级/灰度控制。

说明：本清单只描述“已有代码基础上的改造项”，不包含纯新模块从零实现细节。

## 2. 当前基线（已存在）
1. 已有 M:N 调度器、worker 本地队列、WAITING 协程恢复机制。
2. 已有 TraceId 透传、路由维度基础指标、网关/RPC 结构化日志输出。
3. 已有基础负载均衡抽象（`ILoadBalancer`）与轮询实现（RR）。
4. 已有 HTTP 网关主链路与 runtime/rpc/net 端到端联通能力。

## 3. 需改代码项（P0）

### 3.1 调度器：三级调度 + 工作窃取
目标：
1. 从“worker 单本地队列”升级为“全局队列 + 线程本地队列 + 协程本地队列”。
2. 引入工作窃取，提升空闲线程利用率与任务分布均衡性。
3. 对 IO 唤醒协程进行高优先级回投，降低尾延迟。

建议改动文件：
1. `include/rpc/runtime/scheduler.h`
2. `src/runtime/scheduler.cpp`
3. `tests/unit/scheduler_mn_test.cpp`
4. `tests/unit/io_epoll_integration_test.cpp`

验收关注：
1. 多线程压力测试无死锁、无任务丢失。
2. 空闲线程能稳定窃取繁忙线程任务。
3. QPS 与 P99 相比现基线有可复现提升。

### 3.2 可观测性：Tracing + Metrics + Logging
目标：
1. 将现有 TraceId 透传升级为标准化 Trace/Span 语义（gateway/rpc/net/runtime）。
2. 增加运行时指标：调度队列长度、协程数、IO 等待数、内存水位、重试预算命中。
3. 统一日志字段（`trace_id/span_id/route/service/node/code/latency_ms`）并支持采样。

建议改动文件：
1. `include/rpc/gateway/http_gateway.h`
2. `src/gateway/http_gateway.cpp`
3. `include/rpc/rpc/client.h`
4. `src/rpc/rpc_stub.cpp`
5. `include/rpc/net/reactor_http_server.h`
6. `src/net/reactor_http_server.cpp`
7. `include/rpc/runtime/runtime.h`
8. `src/runtime/runtime_stub.cpp`
9. `tests/unit/w10_gateway_rpc_trace_metrics_test.cpp`

验收关注：
1. 同一请求可通过 `trace_id + span_id` 串联 gateway/rpc/net 日志。
2. 指标可按路由与服务维度查看请求量、错误率、延迟分位数。
3. 故障排查可从单条请求追踪到具体慢阶段。

### 3.3 负载均衡：自适应 + 熔断降级 + 灰度
目标：
1. 将 RR 升级为基于实时状态的动态加权负载均衡。
2. 接入熔断状态机与降级返回策略。
3. 支持按标签/比例的灰度流量分发。

建议改动文件：
1. `include/rpc/rpc/client.h`
2. `src/rpc/rpc_stub.cpp`
3. `include/rpc/net/net.h`
4. `src/net/net_stub.cpp`
5. `tests/unit/rpc_abstraction_test.cpp`
6. `tests/unit/w7_timeout_retry_test.cpp`

验收关注：
1. 节点权重可按实时状态动态调整。
2. 故障节点进入熔断后流量显著下降，恢复后可半开探测回流。
3. 灰度规则生效且可观测。
