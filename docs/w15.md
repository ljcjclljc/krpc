## 3. 任务落地说明

### 3.1 全链路压测与 Profiling
新增 `w15_fullchain_perf_test`，覆盖：
1. 并发压测：`16` 并发 worker，`1600` 请求总量。
2. 混合负载：`/rpc/invoke` + `/short` + `/slow`。
3. 长连接模型：在可绑定 loopback 场景下，单连接多请求复用（Reactor 模式）。
4. 受限环境回退：无法 bind/listen 时自动切到 in-process 模式，继续输出核心性能指标与剖面。

新增剖面输出：
1. 调度剖面：`enqueue/dequeue` 各队列来源、`idle_switches`、`steal_count`、`state_lock_wait_ns`。
2. 负载均衡剖面：`select_calls`、熔断阻断次数、节点选中分布、LB 锁等待统计。
3. 内存池剖面：分代命中率、高水位、fallback 次数、锁等待统计。
4. 系统调用剖面：`epoll_ctl MOD` 调用/跳过次数、`send/recv` 调用次数（Reactor 模式有效）。

### 3.2 Protobuf schema 与热点路径
1. `krpc.proto` 从 `repeated MetadataEntry` 调整为 `map<string,string>`（字段号保持 `4`）。
2. `krpc.proto` / `raft.proto` 增加：
- `option optimize_for = SPEED;`
- `option cc_enable_arenas = true;`
3. `KrpcChannel` 编码路径改为：
- `ByteSizeLong + SerializeToArray`（预分配输出字符串）
- 线程局部消息对象复用（减少重复构造）
4. 解码路径保留完整结构映射（service/method/payload/metadata/max_retries）。

### 3.3 第一轮性能优化

#### 3.3.1 锁竞争优化（Scheduler）
1. Worker 热路径改为直接使用缓存 `WorkerContext`，移除本地/窃取路径中不必要的全局状态锁读取。
2. 增加状态锁等待统计：`state_lock_wait_ns_total / samples / avg`。
3. 增加调度队列来源统计：
- `enqueue_local/global/fast`
- `dequeue_local_fast/local/global/steal`

#### 3.3.2 系统调用次数优化（Reactor）
1. `Connection` 增加 `write_interest_enabled` 状态。
2. `update_interest()` 在兴趣集未变化时跳过 `epoll_ctl(EPOLL_CTL_MOD)`。
3. 新增 syscall 计数：
- `epoll_mod_calls`
- `epoll_mod_skipped`
- `send_syscalls`
- `recv_syscalls`

#### 3.3.3 负载均衡与内存池剖面
1. LB 运行态统计：`select/feedback` 次数、阻断次数、节点分布、锁等待。
2. 内存池统计新增：`lock_wait_ns_total/samples/avg`。
