## 3. W18 任务落地说明

### 3.1 依赖收敛与接口抽象收尾（跨层耦合清理）
本阶段将跨层调用关系收敛为固定边界，避免“上层直接绕过抽象访问下层实现细节”：
1. Gateway 层仅通过 `rpc/client` 抽象发起调用，不直接触达传输与配置内部实现。
2. Runtime/Net 继续作为执行与传输基础设施，不向业务暴露内部状态对象。
3. Infra 配置、内存池、Raft 等能力通过稳定接口暴露，避免上层直接耦合具体数据结构。
4. 构建层统一收敛到 `rpc_core` 聚合库，减少目标间重复链接和隐式依赖扩散。

收敛结果：
1. 分层边界清晰，跨层改动影响面下降。
2. 后续模块替换（LB、配置、传输实现）可控，不需要大面积改调用方。

### 3.2 最终发布包构建与版本基线冻结（tag）
版本基线：
1. 工程版本基线：`CMakeLists.txt` 中 `project(... VERSION 0.1.0)`。
2. 运行时版本基线：`src/common/version.cpp` 返回 `0.1.0`。
3. 发布冻结建议标签：`v0.1.0-w18`（或团队约定等价命名）。

可复现构建流程（建议作为 release runbook 固化）：
```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
ctest --test-dir build --output-on-failure
```

发布归档建议：
```bash
mkdir -p dist
cp build/rpc_gateway_app dist/
sha256sum dist/rpc_gateway_app > dist/rpc_gateway_app.sha256
tar -czf dist/rpc_gateway_v0.1.0-w18.tar.gz dist/rpc_gateway_app dist/rpc_gateway_app.sha256
```

基线冻结（tag）建议步骤：
```bash
git tag -a v0.1.0-w18 -m "W18 baseline freeze: reproducible release"
git push origin v0.1.0-w18
```
