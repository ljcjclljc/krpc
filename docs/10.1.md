# 10.1 变更说明（Runtime 收敛 + 网关协程化 + 真实 TCP 下游）

## 1. 目标
本次修改围绕一个核心目标：
1. 将 W2~W6 能力收敛为统一 runtime 包装层，对外提供稳定接口。
2. 让网关请求处理真正进入协程调度链路执行。
3. 去除旧的 `invoke_stub` 模拟下游路径，RPC 默认改为真实 TCP 调用。
4. Hook 开关改为手动控制（默认关闭）。

## 2. 总体结果
本次实现后：
1. runtime 对外统一暴露：`start/stop`、`submit`、`io await`、`deadline context`、`hook enable`。
2. 网关请求在 `runtime::submit` 协程任务中执行。
3. RPC 下游调用默认走 `socket/connect/send/recv`（`invoke_tcp`）。
4. Hook 不再强制开启，改为通过 `ReactorHttpServerOptions.enable_runtime_hook` 手动控制。

## 3. 代码改动明细

### 3.1 Runtime 包装层（新增统一外观）
文件：
- `include/rpc/runtime/runtime.h`
- `src/runtime/runtime_stub.cpp`

关键改动：
1. `runtime.h` 从占位入口扩展为 Facade：
- `start_runtime/stop_runtime/runtime_ready`
- `submit/wait_runtime_idle`
- `await_io`
- `create_deadline_context/current_deadline_context`
- `ScopedDeadlineContext`
- `set_runtime_hook_enabled/runtime_hook_enabled`
- `ScopedRuntimeHookEnable`
- 兼容入口 `init_runtime/shutdown_runtime`
2. `runtime_stub.cpp` 实现 `RuntimeHost`（持有 `CoroutineScheduler + IOManager`），并在启动时绑定 Hook IOManager。
3. 提供统一封装，避免业务模块直接依赖 runtime 内部细节。

### 3.2 Gateway deadline 接口切换到 runtime 外观
文件：
- `src/gateway/gateway_stub.cpp`
- `src/gateway/http_gateway.cpp`

关键改动：
1. 将直接使用 `RequestContext` 的代码替换为 runtime 包装接口：
- `create_deadline_context`
- `set_deadline_context_value`
- `ScopedDeadlineContext`
2. 降低 gateway 对 runtime 内部实现（request_context 细节）的耦合。

### 3.3 Reactor 接入 runtime 调度执行
文件：
- `include/rpc/net/reactor_http_server.h`
- `src/net/reactor_http_server.cpp`

关键改动：
1. `ReactorHttpServer` 在 `start()` 时检查 runtime：
- 若未启动则自动 `start_runtime()`，并记录 `owns_runtime_`。
- `stop()` 时按所有权决定是否 `stop_runtime()`。
2. `process_request()` 改为通过 `runtime::submit` 投递协程任务执行网关逻辑，主 worker 通过 `future` 等待结果。
3. 请求执行路径接入 `ScopedRuntimeHookEnable(...)`。

### 3.4 Hook 改为手动开关（默认关闭）
文件：
- `include/rpc/net/reactor_http_server.h`
- `src/net/reactor_http_server.cpp`

关键改动：
1. 新增配置：`ReactorHttpServerOptions.enable_runtime_hook`（默认 `false`）。
2. `process_request()` 不再硬编码 `ScopedRuntimeHookEnable(true)`，改为按配置值开启/关闭。

### 3.5 去除旧 stub 路径，RPC 默认走真实 TCP
文件：
- `include/rpc/net/net.h`
- `src/net/net_stub.cpp`
- `src/rpc/rpc_stub.cpp`
- `src/gateway/http_gateway.cpp`
- `include/rpc/gateway/http_gateway.h`
- `include/rpc/net/reactor_http_server.h`

关键改动：
1. 删除 `invoke_stub` 声明与实现。
2. `net` 保留并增强 `invoke_tcp`：
- 解析 `host:port`
- 应用 `derive_effective_timeout`
- 执行 `socket/connect/send/recv/close`
- 返回错误语义（如 connect/send/recv timeout/failed）
3. `DefaultRpcClient::invoke` 统一调用 `invoke_tcp`。
4. 移除网关层 transport 模式配置与请求头覆盖逻辑（不再支持 stub/tcp 双模式切换）。

## 4. 测试改动
文件：
- `tests/unit/w10_gateway_rpc_trace_metrics_test.cpp`
- `tests/unit/w7_timeout_retry_test.cpp`
- `tests/unit/io_epoll_integration_test.cpp`
- `tests/unit/rpc_abstraction_test.cpp`

关键改动：
1. 为依赖 RPC 下游调用的测试补充本地 TCP Echo Backend，避免依赖 `invoke_stub`。
2. 在受限环境（`EPERM/EACCES`）下增加跳过逻辑，避免 CI/Sandbox 因无 socket 权限误报失败。
3. `w10` 保留端到端链路验证，并验证真实 TCP 路径命中。

## 5. 新调用链（当前）
用户请求路径：
1. `Client -> ReactorHttpServer (accept/read/parse)`
2. `Reactor worker -> runtime::submit(协程任务)`
3. `HttpGateway::handle -> /rpc/invoke -> IRpcClient::invoke`
4. `DefaultRpcClient::invoke -> net::invoke_tcp`
5. `invoke_tcp -> socket/connect/send/recv`
6. 回包映射 HTTP 并写回客户端

## 6. 行为变化与影响
1. 正向变化：
- 请求执行进入 runtime 协程调度链路，W2~W6 能力开始在主链路承载。
- 下游调用从模拟变为真实网络行为，更接近生产语义。
2. 兼容变化：
- 不再存在 stub 兜底，运行环境需具备可用 TCP 后端或对应测试后端。
- Hook 默认关闭，需显式配置 `enable_runtime_hook=true` 才启用 Hook 路径。

## 8. 后续建议
1. 增加配置化 backend 地址管理（替代测试内硬编码 endpoint）。
2. 在 runtime 层增加 submit 策略参数（如 HookMode），继续减少业务层对 Hook 细节感知。
3. 为 `invoke_tcp` 增加连接池与健康检查，降低每次新建连接成本。
