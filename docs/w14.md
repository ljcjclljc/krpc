# W14 帮助文档（09-30 ~ 10-06）

## 1. 目标与范围
W14 目标是补齐内存治理与日志排障能力，覆盖三项核心任务：
1. 将固定槽位思路升级为分代式自适应内存池（短生命周期/长生命周期分池）。
2. 实现热扩容/冷缩容，并沉淀高水位与分代命中指标。
3. 增加泄漏检测、越界检查（ASAN 模式）与结构化日志采样策略。

## 3. 设计说明

### 3.1 分代式自适应内存池
`AdaptiveMemoryPool` 采用双池模型：
1. `ShortLived`：短生命周期请求内存。
2. `LongLived`：长生命周期或较大块内存。

核心机制：
1. 每代池独立维护 `slot_size/initial_slots/max_slots/free_list`。
2. 分配命中分池时返回池内块；超出槽位大小时走 fallback 分配。
3. 每次分配记录请求字节、分代、allocation_id，用于泄漏与越界审计。

### 3.2 热扩容与冷缩容
1. 热扩容触发条件：`in_use/capacity >= hot_expand_utilization`。
2. 冷缩容触发条件：连续低利用率 tick 达到 `cold_shrink_idle_ticks`，且容量高于 `initial_slots`。
3. 每代池分别记录 `hot_expand_count/cold_shrink_count`。

### 3.3 越界检查与泄漏检测
1. 每个分配块带 `front_guard + tail_guard`。
2. 释放时校验 guard，越界/破坏计入 `guard_violations`。
3. 通过 `detect_leaks()` 快照输出当前未释放分配（数量、字节、样本记录）。
4. 析构阶段若仍有未释放块，输出 leak 摘要日志。

### 3.4 结构化日志采样策略
新增 `structured_log`：
1. 日志格式：JSON 行（`ts_unix_ms/level/event/trace_id/latency_ms/...`）。
2. 采样策略：按级别采样率（debug/info/warn/error）+ 慢请求强制采样。
3. 慢请求阈值：`slow_request_threshold_ms`。
4. 采样统计：`emitted_total/dropped_total/forced_by_slow_total`。

## 6. 验收映射

### 6.1 内存利用率与扩缩容稳定
对应 `w14_memory_pool_observability_test`：
1. 并发分配/释放稳定，最终 `active_allocations == 0`。
2. 高负载触发热扩容（容量上升）。
3. 低负载触发冷缩容（容量回落）。
4. 分代命中率（short/long）均可观测且大于 0。

### 6.2 泄漏、越界与慢请求定位
对应测试与日志：
1. 越界写触发 guard 失败并更新 `guard_violations`。
2. `detect_leaks()` 可识别未释放块。
3. 慢请求触发强制采样，日志含 `slow_stage` 与阶段耗时。
