## 3. 核心设计说明

### 3.1 协程状态机
状态定义在 `CoroutineState`：
- `READY`：可被调度执行
- `RUNNING`：当前正在执行
- `TERM`：执行结束（正常结束或异常结束）

状态流转规则：
1. 新建协程时为 `READY`
2. `resume()` 后进入 `RUNNING`
3. 协程主动 `yield()` 后从 `RUNNING -> READY`
4. 协程函数执行完毕后从 `RUNNING -> TERM`

### 3.2 `resume/yield` 切换机制
- Windows：使用 Fiber（`CreateFiberEx` / `SwitchToFiber`）
- 非 Windows：实现了 `ucontext` 路径（`swapcontext` 等）

关键约束：
1. 仅允许 `READY` 状态协程被 `resume()`
2. 仅允许当前运行中的协程执行 `yield()`
3. 当前实现不支持“嵌套 resume”（一个协程内部再直接 `resume` 另一个协程）

### 3.3 生命周期管理与回收
`CoroutineScheduler` 负责：
1. 调度 READY 队列
2. 执行后根据状态重新入队或回收
3. 回收 TERM 协程对象
4. 复用回收后的协程 ID

核心计数：
- `alive_count()`：当前仍存活（未回收）协程数
- `completed_count()`：累计完成并回收的协程数

## 4. 对外接口速查

### 4.1 `Coroutine`
- `resume()`：从调度器或调用方切入协程执行
- `yield()`：当前协程主动让出执行权
- `state()`：查询当前状态
- `id()`：查询协程 ID
- `Coroutine::yield_current()`：当前协程快捷让出接口

### 4.2 `CoroutineScheduler`
- `schedule(callback, stack_size)`：创建协程并加入 READY 队列
- `run()`：循环调度直到 READY 队列清空
- `recycle_terminated()`：主动回收已结束协程
- `alive_count()` / `completed_count()`：运行统计

## 7. 常见问题排查

### 7.1 `Test not available without configuration`
现象：`ctest` 在 Visual Studio 生成器下报缺少配置。  
处理：增加配置参数：
```powershell
ctest --test-dir build -C Debug --output-on-failure
```

### 7.2 协程无法 `yield_current()`
现象：抛出“当前没有运行协程”相关错误。  
原因：在普通线程上下文（非协程回调）调用了 `yield_current()`。  
处理：只在 `Coroutine` 回调体内部调用该接口。

### 7.3 嵌套调度异常
现象：抛出 “Nested coroutine resume is not supported”。  
原因：在一个协程未让出时尝试直接恢复另一个协程。  
处理：通过调度器统一调度，不在协程内部直接嵌套 `resume()`。
