# 模块目录规划

## 目录结构

```text
rpc/
|- CMakeLists.txt
|- cmake/
|  |- ProjectOptions.cmake
|- include/rpc/
|  |- common/      # 公共定义（版本、错误码、工具）
|  |- runtime/     # 协程、调度器、定时器、Hook 的对外头文件
|  |- net/         # Socket、Epoll、连接管理的对外头文件
|  |- gateway/     # HTTP 解析、路由、Servlet 的对外头文件
|  |- rpc/         # KrpcChannel、编解码、负载均衡的对外头文件
|  `- infra/       # 日志、配置、监控、内存池的对外头文件
|- src/
|  |- common/      # common 模块实现：公共工具、版本、错误码实现
|  |- runtime/     # runtime 模块实现：协程/调度/定时器/Hook 核心逻辑
|  |- net/         # net 模块实现：网络事件循环、fd 事件与连接生命周期
|  |- gateway/     # gateway 模块实现：HTTP 网关处理链路与转发逻辑
|  |- rpc/         # rpc 模块实现：服务发现、负载均衡、连接池、调用逻辑
|  `- infra/       # infra 模块实现：日志、配置、指标、内存池等基础设施
|- apps/
|  `- gateway/     # 网关可执行程序入口
|- tests/
|  |- unit/        # 单元测试
|  `- integration/ # 集成测试
|- scripts/        # 构建、压测、辅助脚本
`- third_party/    # 第三方依赖或子模块
```

## 模块职责
- `runtime`：提供协程抽象、M:N 调度、IO 协程调度和 Hook 框架。
- `net`：封装网络事件模型和连接生命周期管理。
- `gateway`：承接 HTTP 请求并转发至 RPC 客户端。
- `rpc`：负责服务发现、负载均衡、连接池与请求调用。
- `infra`：提供日志、配置、指标、内存池等基础能力。
- `common`：放置跨模块可复用的工具与定义。

## 约束
- 头文件与实现文件按模块对齐，禁止跨层直接依赖内部实现。
- 新增模块时，必须同步更新 `include/`、`src/` 和本规划文档。
- 网关主程序只通过模块公开头文件访问能力。
