# W16 帮助文档（第二轮性能优化 + 24h 稳定性压测 + 参数固化）

## 1. 目标与范围
W16 目标是完成“稳定性治理第二轮闭环”，覆盖三项任务：
1. 第二轮性能优化与 24h 稳定性压测（覆盖 Raft 选主、调度窃取、灰度流量、内存扩缩容）。
2. 故障演练（延迟抖动、丢包、节点上下线、Leader 切换）。
3. 收敛并固化限流/熔断/重试/负载均衡默认参数。

## 3. 关键设计与优化说明

### 3.1 Raft 故障注入与演练能力
新增 `RaftFaultInjectionOptions` 与接口：
1. `configure_fault_injection(...)`
2. `fault_injection_options()`

注入项：
1. `packet_loss_percent`：复制路径丢包率。
2. `replicate_delay_ticks_min/max`：复制延迟抖动（tick 级）。

新增观测：
1. `RaftStatus.replication_delayed`
2. `RaftStatus.replication_dropped`

实现要点：
1. 复制路径抽象为共享 `ReplicationPayload`，减少每个 follower 重复构造复制载荷（第二轮热路径优化）。
2. 延迟复制走 `pending_replications_` 队列，在 `tick()` 中自然回放。
3. 节点下线时清理该节点待回放复制任务，避免无效堆积。

### 3.2 限流默认生效与网关快速失败
`ReactorHttpServer` 增加全局入口限流：
1. 新选项：`rate_limit_qps`、`rate_limit_burst`。
2. 若选项为 `0`，自动读取配置中心默认值（`gateway.rate_limit.qps/burst`）。
3. 在 Reactor 读路径请求入队前执行限流判定，命中即返回 `429 rate_limited`。

新增观测：
1. `ReactorHttpServerStats.rejected_rate_limited_429`（与队列满导致的 429 分开统计）。

### 3.3 熔断/重试/LB 参数收敛与配置兜底
`rpc_stub` 参数解析改为“Header 优先 + Config 默认兜底”：
1. LB：`weight.cpu/mem/qps/latency`、`target_qps`、`target_latency_ms`。
2. Circuit Breaker：`failure_threshold/open_ms/half_open_success/half_open_max_probes`。
3. Retry：`max_retries`、`retry budget` 窗口/比例/最小 token。

默认配置来源：`infra` 快照（`ConfigRepository`），当请求头未显式覆盖时自动生效。

### 3.4 Boost 使用点（W16）
W16 在优化路径继续优先复用 Boost：
1. `boost::container::small_vector`：用于 LB 候选节点索引（小规模热路径减少堆分配）。
2. `boost::algorithm::clamp`：用于 Raft 故障注入参数归一化。
3. 既有 `boost::hash/algorithm/lexical_cast/circular_buffer` 持续用于灰度哈希、参数解析和状态采样。

## 5. W16 验收测试
新增测试：`w16_stability_chaos_tuning_test`

覆盖点：
1. 调度窃取：构造本地队列倾斜，验证 `steal_count/dequeue_steal` 可观测。
2. 内存治理：验证热扩容与冷缩容均发生。
3. 灰度流量：按 `x-gray-percent` 进行用户哈希分流，验证灰度比例落在阈值区间。
4. 参数固化：校验默认配置键存在，并验证重试预算默认值已生效。
5. Raft 故障演练：在丢包 + 延迟抖动 + 节点上下线 + Leader 切换下持续写入并最终一致。

## 6. 24h 稳定性压测运行方式

### 6.1 常规快速验证
```bash
cmake -S . -B build
cmake --build build -j --target w16_stability_chaos_tuning_test
ctest --test-dir build -R "w16_stability_chaos_tuning_test" --output-on-failure
```

### 6.2 长时模式（建议用于 24h 任务）
该测试支持通过环境变量放大压力轮次：
```bash
RPC_W16_LONG_RUN=1 \
RPC_W16_SCHED_TASKS=50000 \
RPC_W16_GRAY_REQUESTS=20000 \
RPC_W16_RAFT_ROUNDS=120000 \
./build/w16_stability_chaos_tuning_test
```

建议执行策略：
1. 在 CI/压测机中按小时分段跑并汇总指标。
2. 以 `leader_changes/replication_delayed/replication_dropped/steal_count` 为核心观测。
3. 每轮结束后检查 `metadata_consistent`、错误率与内存高水位。
